Frequently-asked Questions for Assignment #3

  1. Q: How do I get the words that the user typed as command-line arguments?

    A: You could join the words one at a time, with a space between them:

        text = ''
    for word in sys.argv[1:]:
        text += word + ' '

    Or you can let python do it for you:

        text = ' '.join(sys.argv[1:])

  2. Q: How do I create a two-dimensional array?

    A: It’s just an array of arrays:

        picture = []
    for row_index in range(height):
        picture.append([" "] * width)

  3. Q: I’m confused. Why should I make an array of arrays? Can’t I just make an array of strings?

    A: You could, but it’s a bit awkward to replace one character in a string, because strings are immutable. You would have to make a fresh string using slices:

         picture[y] = picture[y][:x] + symbol + picture[y][x+1:]

    whereas it’s easier with an array of arrays:

         picture[y][x] = symbol

  4. Q: That’s weird. picture[y][x] ? Why not picture[x][y]?

    A: picture is an array of arrays. picture[y] is one row in that array. picture[y][x] is one entry in that row. The [] parentheses work left-to-right.

  5. Q: How do I get the blocks in the first place?

    A: They are in the file matrix_5x7.txt . Open that file, and read its lines. Some lines are 1-character symbols, and other lines are 5-character rows in a picture.

    You could call readlines(), and get all the lines, then walk through this list of lines. Each time you get a length-1 line, advance through the next 7 lines, and collect them into a block.

    Another way is to call readline() many times (note that readline != readlines) !!!. Here is example code that counts the lines in file. It creates an infinite loop (while True:), and exits the loop when a zero-length line is read (this happens at the end of the file).

       n_lines: int = 0
   while True:
       line: str = handle.readline()
       if len(line) == 0:
           break
       n_lines += 1
       line = line.rstrip("\r\n")
   handle.close()

  6. Q: So I have to go through the lines, get one (for the symbol), and then get 7 more (for the matrix). What’s a good way to do that?

    A: The simplest way is to call .readline() the correct number of times:

         while True:
         line: str = handle.readline()
         if len(line) == 0:
             break
         symbol = line.rstrip("\r\n")
         line1 = handle.readline()
         line2 = handle.readline()
         line3 = handle.readline()
         ...

     handle.close()

  7. Q: OK, I’ve just read one symbol, and 7 strings. How do I store these?

    A: This is a perfect case for using a dictionary:

         block: List[str] = [line1, line2, line3, line4, line5, line6, line7]
     matrices[symbol] = block

  9. Q: If I have a 5 x 7 block, how do I figure out where its pixels are, in the picture?

    A: If you know the text_row, text_col of your block, you can use math. Each block is 6 wide, and 8 high (5+1, 7+1).

    You can figure out the block’s top-left pixel:

        top_left_x = text_col * 6
    top_left_y = text_row * 8

    Now, for each entry in the 5 x 7 block, you should add to top_left_x and top_left_y to get the entry’s x and y coordinates:

        pixel_x = top_left_x + x
    pixel_y = top_left_y + y

  10. Q: I’m confused by all those widths and heights.

    A: Remember that you are dealing with letters, and also with pixels.

    text_height and text_width are how many rows and columns your text takes up.

    Example: if the message is "Hello World", then text_height is 1, and text_width is 11.

    Example: if the message is "Hello World How are you", then text_height is 2, and text_width is 13 (there are two rows of text, at most 13 columns wide).

    The picture is made up of pixels. Each letter takes up 5 pixels wide, plus 1, so 13 letters take up 6*12 + 5 = 77 pixels wide. So your picture should be 77 pixels wide.

    Each row of text takes up 7 rows of pixels, plus 1, so 3 rows of text would take up 2*7 + 6 = 20 rows of pixels. So, in that case, your picture would be a 2-dimensional array. It has 20 entries, and each entry is an array of 77 " " spaces.
